# Philosophy

## 기존 AI 어시스턴트

```
질문 → 답변 → 끝
```

경험이 휘발됩니다. 어제 해결한 어려운 버그도, 왜 그 기술을 골랐는지도 남지 않습니다.

## Developer Second Brain

```
질문 → 개발 → 문제 해결 → 기록 가치 판단 → 문서화 → 저장 → 면접·포트폴리오 자산 축적
```

AI는 답변만 하는 것이 아니라, **개발자의 경험을 자산으로 만듭니다.**

## 다섯 가지 목표

1. **Knowledge Preservation** — 개발하며 얻은 경험을 잃지 않는다. (JWT, Redis, React Query, Docker…)
2. **Engineering Documentation** — 기록을 자동으로 정리한다. (Troubleshooting, ADR, Retrospective)
3. **Portfolio Building** — 좋은 경험을 포트폴리오 소재로 축적한다.
4. **Interview Preparation** — 면접 사례를 모은다. STAR로 확장 가능하게.
5. **Long-term Memory** — vault(→ Notion)를 개발자의 Second Brain으로 쓴다.

## Claude의 역할

- Documentation Engineer
- Knowledge Manager
- Technical Mentor
- Interview Assistant

## 흔들리지 않기 위한 기준

기능을 추가할 때마다 스스로 묻습니다:

> **"이 기능이 개발자의 지식 축적에 기여하는가?"**

"노션에 자동으로 넣기"는 목적이 아닙니다. 목적은 언제나 **경험을 자산으로 바꾸는 것**입니다.
이 기준을 통과하지 못하는 기능은, 아무리 편리해 보여도 넣지 않습니다.
