# Hooks (v0.4)

자동 탐지 훅. **제안까지만 자동화하고, 저장은 여전히 사용자 승인이 필요합니다.**

## 왜 필요했나

v0.3까지 판단 규칙(`references/`)은 다 있었는데 **제안이 거의 뜨지 않았습니다.**

스킬은 **모델이 스스로 켜는 것**이고, 그 방아쇠는 스킬 설명(description)입니다. 그런데 이
스킬의 발동 조건은 "개발 문제를 **다 풀고 난 뒤에** 쓰세요"예요. 이건 사용자 요청에 매핑되는
조건이 아닙니다. 요청이 없는 상태에서, 작업이 끝난 시점에 모델이 *자발적으로* "이제 기록
가치를 판단해볼까?" 하고 떠올려야 하는데, **그걸 시켜주는 게 아무것도 없었습니다.**

엔진은 멀쩡한데 시동 거는 장치가 없던 셈이고, 그래서 `/document`로 직접 부를 때만 동작했습니다.
이 훅이 그 빠진 방아쇠입니다.

## 구성

| 파일 | 역할 |
|------|------|
| `detect-on-stop.sh` | Stop 훅 본체. 값싼 게이트 → 통과 시 모델에게 판단을 지시 |
| `hooks.json` | 플러그인 설치용 훅 등록 (`${CLAUDE_PLUGIN_ROOT}` 사용) |

심링크 개발 설치는 `hooks.json`을 읽을 플러그인 매니페스트가 없어서, `scripts/install-dev.sh`가
`scripts/hook-wiring.py`로 `~/.claude/settings.json`에 직접 등록합니다.

## 동작

```
개발 종료 (Stop)
   │
   ▼
값싼 게이트 (detect-on-stop.sh)  ← "판단할 가치가 있나?" 만 판정
   │  편집 ≥1 또는 Bash ≥8, 그리고 메시지 ≥12개
   ▼
decision: block + reason → 모델이 탐지 엔진 실행
   │
   ▼
references/ 규칙으로 판단  ← "기록할 가치가 있나?" 는 여기서만
   │
   ▼
가치 있으면 → "기록할까요?" 제안 / 아니면 조용히 종료
```

### 역할 분리가 핵심

훅은 **기록 가치를 절대 판단하지 않습니다.** "작업이 실제로 일어났나?"만 봅니다. 가치 판단은
`references/`에 단 하나로 남아야 하고, 훅이 그걸 흉내내면 규칙이 두 군데로 갈라집니다.

그래서 훅이 발동해도 모델에게 주는 지시에는 이렇게 못박아뒀습니다 — *"이 훅은 작업이
일어났다는 것만 감지했을 뿐, 그 작업이 가치 있다고 판단한 게 아니다. 조용히 넘어가는 것이
정상이고 가장 흔한 결과다."*

## 임계값

`detect-on-stop.sh` 상단에 상수로 모여 있습니다.

| 상수 | 값 | 의미 · 정한 이유 |
|------|-----|------------------|
| `MIN_EDITS` | 1 | 파일이 실제로 바뀜 — "작업이 일어났다"의 가장 강한 신호 |
| `MIN_BASH` | 8 | 편집 없이 셸만 두드리는 **순수 디버깅 세션**을 놓치지 않기 위함 |
| `MIN_MESSAGES` | 12 | 너무 짧은 대화(한 줄 읽기, 중단된 시작)를 걸러냄 |

전부 **느슨하게** 잡았습니다. 진짜 판단은 엔진이 하므로, 이 게이트는 "개발 작업이 아예 없는
세션"만 거르면 됩니다. 값을 올리면 방해는 줄지만 놓치는 제안이 늘어납니다.

`MIN_EDITS`와 `MIN_BASH`는 **OR**입니다 (편집이 있거나 셸을 많이 썼거나). `MIN_MESSAGES`는
**AND**로 함께 걸립니다.

### 트랜스크립트를 읽는 방식

두 가지가 측정에 영향을 줍니다. 둘 다 실제 트랜스크립트를 뜯어보고 잡은 것입니다.

- **줄 수가 아니라 메시지 수를 셉니다.** JSONL에는 대화 외에 스냅샷·훅 기록·첨부 같은 관리용
  줄이 섞여 있습니다. 실제 세션에서 **564줄 vs 실제 메시지 309개** 였습니다. 줄 수로 세면
  게이트가 자기가 검사한다고 말한 것보다 조용히 느슨해집니다. `.message`가 있는 항목만 셉니다.
- **슬럽하지 않고 스트리밍합니다.** `jq -s`로 전체를 메모리에 올리면 149MB 트랜스크립트에서
  **RSS 516MB**가 나왔습니다. 이 훅은 매 Stop마다 도는데 그건 과합니다. 줄 단위로 흘려보내면
  세션 길이와 무관하게 **3.3MB**로 평평하게 유지됩니다 (실제 2MB 트랜스크립트 기준 0.05초).
- 메시지의 `content`는 배열이 아니라 **문자열**인 경우도 있습니다(실제 세션에 13건). 이 경우
  툴 추출은 건너뛰고 메시지 수에는 포함합니다.

## 안전장치

지식 기록 도구가 사용자 세션을 망가뜨리면 안 됩니다. 제안을 한 번 놓치는 건 `/document`로
복구되지만, 훅이 터지는 건 복구가 안 됩니다. 그래서 **예상 못 한 모든 상황에서 조용히 exit 0**
합니다.

- **루프 방지** — Stop을 막으면 모델이 다시 돌고 Stop이 또 발생합니다. `stop_hook_active`를
  검사해 두 번째부터는 즉시 빠집니다. 이게 없으면 세션이 끝나지 않습니다.
- **세션당 1회** — 발동 전에 마커 파일을 먼저 씁니다. 이후 단계가 실패해도 루프가 안 생깁니다.
  (`references/interruption-policy.md`의 세션당 제안 상한과 같은 규칙)
- **stderr 억제** — 입력이 깨져도 에러 메시지가 사용자 터미널에 새지 않습니다.
- **jq 없으면 침묵** — 추측하지 않고 그냥 빠집니다.

## 끄기

```bash
export SECOND_BRAIN_HOOK_DISABLED=1   # 런타임에 비활성화
./scripts/install-dev.sh --no-hook    # 설치할 때 훅만 제외
./scripts/uninstall-dev.sh            # settings.json에서 제거
```

## 테스트

```bash
./tests/hook-gate.sh
```

게이트 판정(발동/침묵) 18개 케이스를 검증합니다. `tests/fixtures/`가 *엔진의 판단*을
검증한다면, 이쪽은 그 아래 계층인 *훅의 게이트*를 검증합니다.

## 상태 파일

세션당 발동 여부 마커가 쌓입니다.

- 기본값: `~/.claude/second-brain-state/`
- 변경: 환경변수 `SECOND_BRAIN_STATE_DIR`

빈 파일이라 용량은 무시할 수준이고, 지워도 안전합니다 (해당 세션에서 다시 한 번 발동할 수 있게
될 뿐입니다).
